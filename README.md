## 先搞清楚一件事：DC6 和 DC9 是什么

**DC6**（内部代号 USCA_6），官方叫做 CN2 GIA ECOMMERCE，缩写 CN2 GIA-E。机房托管在洛杉矶的 ZenLayer 数据中心，带宽起步 2.5Gbps，最高可到 10Gbps。

**DC9**（内部代号 USCA_9），搬瓦工最早引进的 CN2 GIA 线路，向中国电信直接拉的专线，带宽固定 1Gbps，近年来宿主机已经完成了向 AMD EPYC + NVMe 的升级。

两个机房都在美国洛杉矶，都走三网 CN2 GIA——电信直达，移动和联通也通过电信 CN2 GIA 中转。从北京测过来的延迟大概在 150–200ms 左右，相差不超过 10ms，基本可以忽略不计。

## DC6 和 DC9 最核心的区别：带宽和线路结构

差异就集中在两点，其他都是细节。

**第一：带宽差距很大。**

DC6 起步 2.5Gbps，峰值 10Gbps。DC9 固定 1Gbps。光从带宽数字看，DC6 就是 DC9 的 2.5 倍起步。晚高峰的时候这个差距会体现得比较明显，特别是大文件传输或者有大量并发请求的场景。

**第二：线路结构不完全一样。**

DC9 是搬瓦工直接向中国电信采购的带宽，线路纯度很高，三网去回程几乎都强制走 CN2 GIA。老玩家喜欢叫它"传家宝"，指的就是这条线路够干净、够直接。

DC6 用的是 ZenLayer 的 HIA（High Internet Access）混合网络——对电信走 CN2 GIA，对移动走 CMIN2（中国移动高端线路），对联通走 CUP/AS9929 优化路由。这就是为什么很多人测试下来，DC6 对移动和联通的表现比 DC9 更好的原因。另外最近搬瓦工也给 DC9 加上了 CMIN2 回程，所以两者在移动方向上的差距正在缩小。

丢包率方面，用下来 DC6 和 DC9 都非常低，晚高峰几乎都是 0 丢包的水平，DC6 稍微好一点点，但差距不大到需要特别纠结。

## 按运营商怎么选

这部分是很多文章没说清楚的。

**电信用户**：DC6 和 DC9 在电信方向上的差异最小，两者都是直达 CN2 GIA，路由纯度都很高。预算够选 DC6（带宽更大），预算有限选 DC9 足够。

**联通用户**：DC6 有 CUP/AS9929 优化，联通方向表现更稳。优先选 DC6。

**移动用户**：DC6 的 CMIN2 回程对移动用户帮助最大，特别是晚高峰跨网 QoS 问题，DC6 处理得更好。移动用户几乎应该直接选 DC6。

换一个角度说——如果你是做建站、大文件传输、或者同时服务电信/联通/移动三类用户，DC6 因为带宽大、三网适配更均衡，是更合适的机房。如果你的用户几乎全是电信，对线路纯度有偏执，DC9 的电信直连体验也完全够用。
